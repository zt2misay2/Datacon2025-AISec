我们继续与当局和安全社区合作，进行技术分析以确定IOC并了解恶意软件在初始入侵后的行为。到目前为止，我们已经确认了以下行为：入侵载体使用了OpenSLP漏洞，可能是CVE-2021-21974（仍需进一步确认）。日志显示用户dcui参与了入侵过程。加密过程中，恶意软件部署了一个位于/tmp/public.pem中的公钥，并专门针对虚拟机文件（如.vmdk、.vmx等）进行加密。此外，恶意软件试图通过杀死VMX进程来关闭虚拟机以解锁文件，但此功能并不总是按预期工作，导致文件保持锁定状态。恶意软件还创建了一个argsfile，用于存储传递给加密二进制文件的参数（例如跳过的MB数、加密块中的MB数、文件大小）。没有证据表明数据外泄发生过。在某些情况下，文件加密可能部分失败，从而允许恢复数据。土耳其安全研究员Enes Sönmez (@enes_dev) 已经记录了一种VMDK文件恢复的方法，该方法在他的博客(https://enes.dev/) 中进行了描述。我们和其他安全专家测试了这一程序，在多个受影响的服务器上取得了约2/3的成功率。请注意，执行此程序需要较强的技术能力，自行承担风险，并建议寻求专家的帮助。
在之前的帖子版本中，我们曾假设这次攻击与Nevada勒索软件有关，这是一个错误。目前没有任何材料可以将这次攻击归因于任何特定团体。归因从来都不容易，我们让安全研究人员自己得出结论。ESXi操作系统只能安装在裸金属服务器上。为了识别客户的ESXi安装，我们启动了几项基于自动化日志的举措。由于我们无法逻辑访问客户的服务器，因此我们的行动手段有限。对于已识别的裸金属主机，我们在周五下午发送了电子邮件，警告客户潜在的风险并提供如何减轻风险的信息。同时，我们封锁了互联网与安装ESXi的服务器之间的OpenSLP端口（427）。如果出于某种原因需要使用427端口，客户可以在其管理界面中停用过滤规则。我们还启动了扫描，通过测试是否存在指定主机已被入侵的网页和/或ssh横幅来识别被入侵的主机，并通知受影响的客户。我们的支持团队全力动员，帮助客户保护他们的系统，并在他们受到攻击影响时协助恢复。

此次攻击主要针对7.0 U3i版本之前的ESXi服务器，显然通过OpenSLP端口（427）进行。要检查您的ESXi版本，请参阅您的客户界面上的服务器页面，或直接查看系统本身的ESXi界面。对于使用ESX-i的裸金属客户，我们强烈建议紧急采取以下措施：禁用服务器上的OpenSLP服务或将访问限制为仅受信任的IP地址 (https://kb.vmware.com/s/article/76372)，并将您的ESXi升级到最新的安全补丁。之后，确保您的数据已备份（最好是在不可变存储上），只有必要的服务处于活动状态，并通过ACL过滤仅允许受信任的IP地址访问。同时，监控您的系统是否有任何异常行为。使用VMware私有云的客户不受影响，因为设计上SSL网关会阻止外部访问此端口（OpenSLP 427）。对于我们的公共云客户，由于不依赖于ESXi，因此没有发现风险。OVHcloud产品组合中的其他产品也不受此次勒索软件活动的威胁。
马蒂亚·坎帕尼亚诺

勒索软件，即勒索恶意软件的简称，是一种阻止用户访问其系统或个人文件并要求付款以恢复访问权限的恶意软件。最早的勒索软件变种出现在20世纪80年代末，当时要求通过普通邮件寄送付款。如今，勒索软件作者要求通过加密货币或信用卡支付。（Malwarebytes）

勒索软件不仅对个人用户构成严重威胁，也对企业网络环境构成严重威胁，因为它允许网络犯罪分子在短时间内获得大量资金。虽然一些勒索软件变种展示了强大的编程技能和高度复杂性，但勒索软件分发平台使得即使没有开发背景的人也能进行勒索软件活动。在浏览暗网时，不难发现推广勒索软件即服务（Ransomware-as-a-Service）平台的可疑网站。

这些恶意行为者提供付费服务，就像合法的云提供商一样，包括开发、技术支持、定制仪表板等，而无需攻击者了解任何编程语言。运行这些平台的网络犯罪分子为攻击者提供了他们所需的一切，包括二进制文件、文档、详细说明和技术支持。

勒索软件攻击往往成功的原因在于企业组织在其补丁和配置管理策略上存在安全漏洞。大多数组织仅在演示环境中测试后才应用更新和补丁，这可能会为攻击者成功利用最新补丁本应修复的漏洞创造机会。

最近的勒索软件变种如Ryuk，通常通过大型僵尸网络如Emotet或TrickBot传播，变得更为复杂且致命。2019年12月13日，新奥尔良市遭受Ryuk勒索软件攻击后被迫宣布进入紧急状态。感染迅速蔓延，以至于该市不得不命令所有员工关闭计算机并断开Wi-Fi连接。

另一起近期的勒索软件攻击导致美国海岸警卫队基地停运约30小时，据报道是由一名员工打开了一封受感染的电子邮件触发的。

许多人肯定还记得WannaCry勒索软件爆发事件，它在2017年5月短短时间内成功感染了大量计算机。然而，并非所有人都知道微软在这一广泛感染开始之前几个月就已经发布了修补SMB漏洞的补丁，但大多数个人和企业用户尚未应用此补丁。

如果大多数Windows用户早些时候应用了补丁，WannaCry爆发可能不会像实际发生的那样严重。

你应该支付赎金吗？这是一个非常有争议的话题。政府和执法机构大多建议不要支付。一个高效且安全的备份策略可以最小化与勒索软件感染相关的风险。如果定期创建备份副本并安全存储（包括离线存储），受害者用户/组织就有更大的机会在不遭受过度损害的情况下解决这种情况。

然而，有时这个问题的答案并不那么明确。很大程度上取决于被勒索的信息对组织的价值以及组织能承受多长时间的停机时间。即使有一个高效且安全的备份策略（并定期测试！），仍有可能出现某些情况下组织无法承受数据丢失或服务器和工作站停机的情况，即使只是短暂的时间。

备份通常是按固定计划进行的。对于拥有大型网站作为主要业务的公司来说，不能承受例如一天甚至一小时内完成的交易丢失，因为这可能意味着损失数百万美元。此外，如果一个组织的业务依赖于专有信息并且这些信息被勒索，该组织似乎别无选择只能支付。

然而，支付赎金并不总能保证被勒索的文件会被解密。许多勒索软件攻击者会在收到相关比特币付款后提供解密密钥，而其他人则不会。有时由于错误或其他技术问题，提供的解密密钥无法成功解密所有被勒索的文件。即使解密过程有效，也常常非常缓慢且不可靠。

无论攻击者及其解密工具是否释放您的文件，这绝对是你组织不想陷入的情况。

组织可以采取两种不同的方法来防止或缓解这种威胁：被动方法和主动方法，这两种方法并不相互排斥。

被动方法
1. 如果一个或多个客户端被攻破，立即将其从网络中断开。
2. 如果您的公司使用云存储，请断开受感染客户端的连接，以防止它们同步到云端。同样的原则适用于文件服务器/共享。
3. 从已知干净的备份中恢复受损系统和数据。需要实施并测试全面的备份和灾难恢复策略。理想情况下，至少一个备份副本应存储在安全的离线/异地位置，因为一些勒索软件变种会攻击并破坏其在企业网络中找到的任何备份。人为因素在信息安全中至关重要，粗心的一次点击就可能在几秒钟内损坏组织的网络。主动、正式且有效的备份策略对于缓解勒索软件威胁至关重要。
4. 准备用已知干净的黄金镜像重新安装受感染的机器。
5. 在确保受感染机器运行干净且可信的配置后，再将其重新连接到网络。监控一切以确保成功清除。

**01 CVE-2023-31039 Apache bRPC Code Execution Vulnerability**
Threat Level: High
Vulnerability Description:
On May 8, 2023, the Hua Yun An Si Jing security team monitored that Apache officially released a security update, disclosing a vulnerability in versions of Apache bRPC prior to 1.5.0. Apache bRPC is an open-source, industrial-grade C++ RPC framework, commonly used in high-performance systems such as search, storage, machine learning, advertising, and recommendations. An unauthenticated attacker could exploit this vulnerability to modify the ServerOptions pid_file parameter when the bRPC server starts, leading to arbitrary code execution with bRPC process permissions.
