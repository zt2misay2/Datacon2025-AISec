对 Windows 7 的支持已于2020年1月结束。2020年1月14日之后，微软不再为安装了 Windows 7 的 PC 提供安全更新或支持。如果您在支持结束后继续使用 Windows 7，您的 PC 仍然可以工作，但会更加脆弱。即使公司本身不使用 Windows 7，通过供应链、合作伙伴和远程员工使用该系统也会使组织面临更高的风险。一些最严重的数据泄露事件发生在那些在网络安全上投入数百万并内部遵循了所有“最佳实践”的公司中。这是因为薄弱环节可能存在于外部，如仍在使用过时系统的供应商或外包合作伙伴。无论黑客最初是如何获得访问权限的，如果敏感信息被泄露，您的公司可能会受到牵连。这意味着仅仅关注公司的内部安全已经不够了。您还必须解决第三方和供应商的风险状况。此外，大多数公司允许员工通过 VPN 或其他客户端工具从员工计算机远程访问企业资源，这又带来了一整套新的漏洞。为解决这个问题，企业必须使其网络安全计划能够减少不必要的实用程序和功能的启动；拒绝不可信可执行文件的启动；限制高风险应用程序的不可接受行为；隔离对端点部分的访问和/或更改；降低可疑应用程序造成损害的可能性；解锁以允许合法使用高风险功能。  
Currently, many online analysis articles focus on maintaining processor speed, which has become a higher priority than security considerations. Even the repair solutions from manufacturers are designed to avoid affecting device performance. Whenever a vulnerability is exposed, the speed at which vendors release patches is often criticized, and this situation is no different. For these two vulnerabilities, cloud users and IT administrators, in particular, need to pay attention and promptly apply the fixes.  
The impact of the patches on device performance has already been evident. Although most ordinary desktop users and gamers have not noticed any delay, slowdown, or performance degradation, users running IO or system call-intensive software, such as databases on backend servers, might notice the effect of the patched CPU.  
2023年7月24日，AMD发布了一份安全公告，披露了其Zen2计算机处理器微架构中的一个漏洞。该漏洞被AMD命名为“跨进程信息泄露”，也被称为“Zenbleed”。该漏洞被标记为CVE-2023-20593，并被AMD评为中等严重程度。此问题允许攻击者在特定情况下访问CPU处理的敏感信息。所有运行在基于AMD Zen2处理器上的软件，包括虚拟机、沙箱、容器和进程，都会受到影响。利用此漏洞的软件可能已经存在，预计基于此漏洞的攻击很快就会发生。
